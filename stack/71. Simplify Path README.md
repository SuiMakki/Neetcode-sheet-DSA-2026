# 71. Simplify Path

## Problem Summary

Given an absolute Unix-style path, transform it into the simplified canonical path.

Rules:
- `.` means current directory and should be ignored.
- `..` means go up to the parent directory.
- Multiple slashes are treated as a single slash.
- The simplified path must start with `/` and have no trailing slash unless it is root.
- Directory names like `...` are valid names and should not be interpreted as special.

Example:
- Input: `/home//foo/`
- Output: `/home/foo`

---

## Your Implementation
The local solution file is:
- `71. Simplify Path.java`

Your Java approach uses a stack to process each path component:
- Split the input by `/`.
- For each component:
  - ignore empty strings and `.`
  - if `..` and stack is not empty, pop the last directory
  - otherwise, push valid directory names onto the stack
- Finally, build the canonical path from the stack contents.

### Your Java code
```java
class Solution {
    public String simplifyPath(String path) {
        Stack<String> stack = new Stack<>();
        String[] arr = path.split("/");
        for(String a : arr){
            if(a.equals("..") && !stack.isEmpty()){
                stack.pop();
            }
            else if(!a.equals("") && !a.equals(".") && !a.equals("..")){
                stack.push(a);
            }
        }
        StringBuilder sb = new StringBuilder();
        for(String s : stack){
            sb.append("/");
            sb.append(s);
        }
        return sb.length() == 0 ? "/" : sb.toString();
    }
}
```

### Alternative stack implementation
```java
public class Solution {
    public String simplifyPath(String path) {
        Stack<String> stack = new Stack<>();
        String[] paths = path.split("/");

        for (String cur : paths) {
            if (cur.equals("..")) {
                if (!stack.isEmpty()) stack.pop();
            } else if (!cur.equals("") && !cur.equals(".")) {
                stack.push(cur);
            }
        }

        return "/" + String.join("/", stack);
    }
}
```

This version is equivalent in behavior. It uses the same stack-based logic, but it rebuilds the result using `String.join("/", stack)` instead of manually appending segments.

### Approach, time, and space
- Approach: split the path, use a stack to simulate directory navigation, and rebuild the result.
- Time Complexity: O(n), where n is the length of the path string.
- Space Complexity: O(n) in the worst case, for the stack and the split components.

## 1. Brute Force Approach

### Idea
One possible approach is to iterate through the path and manually parse components, handling `.` and `..` while building a new path string.

### Brute Force Code (Java)

```java
class Solution {
    public String simplifyPath(String path) {
        String[] components = path.split("/");
        List<String> result = new ArrayList<>();
        for (String part : components) {
            if (part.equals("") || part.equals(".")) continue;
            if (part.equals("..")) {
                if (!result.isEmpty()) result.remove(result.size() - 1);
            } else {
                result.add(part);
            }
        }
        StringBuilder sb = new StringBuilder();
        for (String dir : result) {
            sb.append('/');
            sb.append(dir);
        }
        return sb.length() == 0 ? "/" : sb.toString();
    }
}
```

### Brute Force Complexity
- Time Complexity: O(n)
- Space Complexity: O(n)

---

## 2. Best Approach: Stack

### Why we use a stack
A stack naturally models the directory history as we traverse the path:
- push valid directories
- pop when encountering `..`
- ignore `.` and empty segments

This keeps the solution simple and efficient.

### Why this is the right choice
- It handles repeated slashes naturally after splitting.
- It supports valid directory names such as `...`.
- It avoids reconstructing intermediate paths repeatedly.

## 3. Constraints

- `1 <= path.length <= 3000`
- `path` contains letters, digits, `.`, `/`, and `_`
- `path` is a valid absolute Unix path

These constraints mean a linear scan with O(n) time is ideal and efficient.

## 5. Important Notes for Revision

- Use a stack for directory navigation.
- Ignore `.` and empty segments.
- `..` pops the last directory if possible.
- Rebuild the canonical path by joining stack items with `/`.
- Return `/` if the stack is empty.
