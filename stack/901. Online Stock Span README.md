# 901. Online Stock Span

## Problem Summary

Design a class that keeps track of stock prices and returns the span of the current price.

The span of a price is the number of consecutive previous days where the price was less than or equal to the current price.

Example:
- If the recent prices are [7, 2, 1, 2] and today’s price is 2, the span is 4.

---

## 1. Brute Force Approach

### Idea
For each new price, look backward day by day and count how many consecutive days have price less than or equal to the current price.

### Why it is slow
This approach checks many previous days repeatedly, which makes it inefficient for many calls to next().

### Brute Force Code (Java)

```java
class StockSpanner {
    private List<Integer> prices;

    public StockSpanner() {
        prices = new ArrayList<>();
    }

    public int next(int price) {
        prices.add(price);
        int span = 0;

        for (int i = prices.size() - 1; i >= 0; i--) {
            if (prices.get(i) <= price) {
                span++;
            } else {
                break;
            }
        }

        return span;
    }
}
```

### Brute Force Complexity
- Time Complexity: O(n) per call in the worst case
- Space Complexity: O(n)

---

## 2. Best Approach: Monotonic Stack

### Why we use a stack
We need to quickly find the previous greater element and also maintain the span efficiently.

A stack helps us store pairs of:
- price
- span of that price

When a new price arrives:
- while the previous price is smaller or equal, we add its span to the current span
- then we push the current price with the computed span

### Key Idea
- Keep a stack of pairs: [price, span]
- For each new price, accumulate span while the stack top price is smaller or equal
- Push the current price and span

### Why this is better
- It avoids repeatedly scanning from the start every time
- It gives an efficient O(1) amortized time per call
- It is a classic stack-based pattern

### Optimized Java Solution

```java
import java.util.Stack;

class StockSpanner {
    private Stack<int[]> stack;

    public StockSpanner() {
        stack = new Stack<>();
    }

    public int next(int price) {
        int span = 1;

        while (!stack.isEmpty() && stack.peek()[0] <= price) {
            span += stack.pop()[1];
        }

        stack.push(new int[]{price, span});
        return span;
    }
}
```

### Optimized Complexity
- Time Complexity: O(1) amortized per call
- Space Complexity: O(n)

---

## 3. Example

### Input
[100, 80, 60, 70, 60, 75, 85]

### Output
[1, 1, 1, 2, 1, 4, 6]

### Explanation
- Each call to next() returns the span for that price.
- The stack keeps track of previous smaller or equal prices efficiently.

---

## 4. Constraints

- 1 <= price <= 10^5
- At most 10^4 calls will be made to next()

These constraints make the stack solution ideal because it handles repeated calls efficiently.

---

## 5. Important Notes for Revision

- This problem is a great example of using a monotonic stack.
- We store the price and its span together.
- The stack allows us to reuse previous information instead of recomputing everything.
- This is a very useful pattern for interview preparation.
