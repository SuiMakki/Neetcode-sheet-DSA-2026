# 735. Asteroid Collision

## Problem Summary

We are given an array of integers representing asteroids moving in a row. Each asteroid has:

- a size equal to its absolute value
- a direction determined by its sign
  - positive: moving right
  - negative: moving left

When two asteroids collide, the smaller one explodes. If both are the same size, both explode.

---

## 1. Brute Force Approach

### Idea
A simple way is to simulate the process directly:

- Start with the array of asteroids.
- Keep checking whether a collision is possible between neighboring asteroids.
- If two asteroids collide, remove the smaller one or both if equal.
- Repeat until no more collisions happen.

### Why it is slow
This approach can become inefficient because you may need to repeatedly scan and adjust the list, which leads to extra work.

### Brute Force Code (Java)

```java
import java.util.ArrayList;
import java.util.List;

class Solution {
    public int[] asteroidCollision(int[] asteroids) {
        List<Integer> alive = new ArrayList<>();

        for (int x : asteroids) {
            alive.add(x);

            while (alive.size() >= 2) {
                int last = alive.size() - 1;
                int prev = last - 1;

                if (alive.get(prev) > 0 && alive.get(last) < 0) {
                    int left = alive.get(prev);
                    int right = alive.get(last);

                    if (Math.abs(left) < Math.abs(right)) {
                        alive.remove(prev);
                    } else if (Math.abs(left) > Math.abs(right)) {
                        alive.remove(last);
                    } else {
                        alive.remove(last);
                        alive.remove(prev);
                    }
                } else {
                    break;
                }
            }
        }

        int[] result = new int[alive.size()];
        for (int i = 0; i < alive.size(); i++) {
            result[i] = alive.get(i);
        }
        return result;
    }
}
```

### Brute Force Complexity
- Time Complexity: O(n^2) in the worst case
- Space Complexity: O(n)

## 2. Best Approach: Stack

### Why we use a stack
A stack is the best choice because the collisions only matter between the current asteroid and the most recent surviving asteroid.

When we process the array from left to right:

- If the current asteroid is moving left and the top of the stack is moving right, they may collide.
- We keep resolving these collisions using the stack.
- This avoids repeatedly scanning the whole array again and again.

### Key Idea
- Traverse the asteroids from left to right.
- Keep a stack of surviving asteroids.
- If a collision happens, resolve it immediately.
- If the current asteroid survives, push it to the stack.

### Why this is better
- It is faster than the brute force method.
- It processes each asteroid only a small number of times.
- It is very clean and easy to understand for interviews and revision.

### Optimized Java Solution

```java
import java.util.Stack;

class Solution {
    public int[] asteroidCollision(int[] asteroids) {
        Stack<Integer> s = new Stack<>();

        for (int i : asteroids) {
            boolean alive = true;

            while (alive && !s.isEmpty() && s.peek() > 0 && i < 0) {
                if (s.peek() < Math.abs(i)) {
                    s.pop();
                } else if (s.peek() == Math.abs(i)) {
                    s.pop();
                    alive = false;
                } else {
                    alive = false;
                }
            }

            if (alive) {
                s.push(i);
            }
        }

        int[] res = new int[s.size()];
        for (int i = s.size() - 1; i >= 0; i--) {
            res[i] = s.pop();
        }
        return res;
    }
}
```

### Optimized Complexity
- Time Complexity: O(n)
- Space Complexity: O(n)

---

## 3. Example

### Input
asteroids = [5, 10, -5]

### Output
[5, 10]

### Explanation
- The 10 and -5 collide, and 10 survives.
- The 5 and 10 never collide, so both remain.

## 4. Important Notes for Revision

- A positive asteroid moves right.
- A negative asteroid moves left.
- Only a right-moving asteroid and a left-moving asteroid can collide.
- The stack helps us keep track of surviving asteroids efficiently.
- This is a classic stack-based problem from LeetCode.
