# 739. Daily Temperatures

## Problem Summary

You are given an array of daily temperatures. For each day, you need to find how many days you must wait until a warmer temperature appears.

If no warmer day exists in the future, the answer for that day is 0.

Example:
- Input: [73, 74, 75, 71, 69, 72, 76, 73]
- Output: [1, 1, 4, 2, 1, 1, 0, 0]

---

## 1. Brute Force Approach

### Idea
A simple way is to check every day and look ahead until you find a warmer temperature.

For each index i:
- start from i + 1
- keep moving forward until a higher temperature is found
- count how many days were skipped
- if none is found, store 0

### Why it is slow
This approach checks many days repeatedly, so it can become inefficient for large arrays.

### Brute Force Code (Java)

```java
class Solution {
    public int[] dailyTemperatures(int[] temperatures) {
        int n = temperatures.length;
        int[] answer = new int[n];

        for (int i = 0; i < n; i++) {
            for (int j = i + 1; j < n; j++) {
                if (temperatures[j] > temperatures[i]) {
                    answer[i] = j - i;
                    break;
                }
            }
        }

        return answer;
    }
}
```

### Brute Force Complexity
- Time Complexity: O(n^2)
- Space Complexity: O(n)

---

## 2. Best Approach: Monotonic Stack

### Why we use a stack
A stack helps us remember the indices of days we have seen so far.

While moving from left to right:
- if the current temperature is warmer than the temperature at the index stored on the stack top,
  then that previous day can be answered immediately
- we keep popping smaller temperatures because they can never be useful again

### Key Idea
- Traverse the array from left to right.
- Keep a stack of indices whose temperatures are still possible answers.
- For each new temperature:
  - pop all smaller temperatures from the stack
  - the current index becomes the next warmer day for those popped indices
  - push the current index onto the stack

### Why this is better
- It avoids checking the same days repeatedly.
- It gives an efficient O(n) solution.
- It is a very common stack pattern used in interviews.

### Optimized Java Solution

```java
class Solution {
    public int[] dailyTemperatures(int[] temperatures) {
        int[] result = new int[temperatures.length];
        Stack<Integer> stack = new Stack<>();

        for (int i = 0; i < temperatures.length; i++) {
            while (!stack.isEmpty() && temperatures[stack.peek()] < temperatures[i]) {
                result[stack.peek()] = i - stack.pop();
            }
            stack.push(i);
        }

        return result;
    }
}
```

### Optimized Complexity
- Time Complexity: O(n)
- Space Complexity: O(n)

---

## 3. Example

### Input
temperatures = [73, 74, 75, 71, 69, 72, 76, 73]

### Output
[1, 1, 4, 2, 1, 1, 0, 0]

### Explanation
- Day 0 waits 1 day for 74
- Day 1 waits 1 day for 75
- Day 2 waits 4 days for 76
- Day 3 waits 2 days for 72
- Day 6 has no warmer day after it, so answer is 0

---

## 4. Important Notes for Revision

- This problem is a classic example of using a monotonic stack.
- The stack stores indices, not temperatures directly.
- We pop from the stack when we find a warmer temperature.
- The answer for each day is computed using the difference in indices.
- This is a very useful pattern for many interview problems.
