# 136. Single Number

## Problem Summary

You are given a non-empty array of integers where every element appears exactly twice except for one element that appears once.

Your task is to find that single element.

You must solve it in:
- linear time
- constant extra space

Example:
- Input: [2, 2, 1]
- Output: 1

---

## 1. Brute Force Approach

### Idea
A simple way is to scan the array and count how many times each number appears.

For each number:
- count its frequency
- if the count is 1, return it

### Why it is slow
This approach uses extra work because it repeatedly checks the array and may take more time than necessary.

### Brute Force Code (Java)

```java
import java.util.HashMap;

class Solution {
    public int singleNumber(int[] nums) {
        HashMap<Integer, Integer> count = new HashMap<>();

        for (int num : nums) {
            count.put(num, count.getOrDefault(num, 0) + 1);
        }

        for (int num : count.keySet()) {
            if (count.get(num) == 1) {
                return num;
            }
        }

        return -1;
    }
}
```

### Brute Force Complexity
- Time Complexity: O(n)
- Space Complexity: O(n)

---

## 2. Best Approach: Bit Manipulation

### Why we use XOR
The XOR operation has a special property:
- a ^ a = 0
- a ^ 0 = a

So if we XOR all numbers together:
- every repeated number cancels out
- the single number remains

### Key Idea
- Start with result = 0
- XOR every number in the array with result
- The final value is the single number

### Why this is better
- It is very efficient
- It uses constant extra space
- It matches the problem requirement perfectly

### Optimized Java Solution

```java
class Solution {
    public int singleNumber(int[] nums) {
        int res = 0;
        for (int num : nums) {
            res ^= num;
        }
        return res;
    }
}
```

### Optimized Complexity
- Time Complexity: O(n)
- Space Complexity: O(1)

---

## 3. Example

### Input
nums = [4, 1, 2, 1, 2]

### Output
4

### Explanation
- 4 appears once
- 1 and 2 appear twice
- XOR of all values leaves 4

---

## 4. Important Notes for Revision

- This is a classic XOR problem.
- XOR is the best tool here because duplicate values cancel out.
- This solution works in constant space and linear time.
- It is a very important pattern for bit manipulation questions.
