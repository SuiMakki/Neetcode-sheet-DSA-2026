# 191. Number of 1 Bits

## Problem Summary

Given a positive integer `n`, return the number of set bits (`1`s) in its binary representation.

Refer to the local solution file:
- `191. Number of 1 Bits.java`

### Your implementation
The file uses a straightforward bit-counting loop:

```java
class Solution {
    public int hammingWeight(int n) {
        int count = 0;
        while (n > 0) {
            if (n % 2 == 1) {
                count++;
            }
            n = n / 2;
        }
        return count;
    }
}
```

This method examines each bit by dividing the number by 2 and counting the remainder. It correctly counts all set bits and is easy to understand.

### Approach, time, and space
- Approach: direct bit scan using division by 2 and checking the least significant bit.
- Time Complexity: O(b), where b is the number of bits in the number. For a 32-bit integer, this is at most 32 steps, so it behaves like O(1) in practice.
- Space Complexity: O(1), because it uses only a constant number of variables.

### How this compares
- Your implementation is a direct bit-scanning approach.
- It examines every bit until the number becomes zero.
- The brute force bit-scan version below is essentially the same in time and space.
- The optimized bit trick below can be faster when the number has very few `1` bits, because it only loops once per set bit.

Example:
- Input: `n = 11`
- Output: `3`
- Explanation: binary `1011` has three set bits.

## 1. Brute Force Approach

### Idea
Convert the number to binary and count the number of `1` characters, or repeatedly examine each bit.

### Brute Force Code (Java)

```java
class Solution {
    public int hammingWeight(int n) {
        int count = 0;
        while (n != 0) {
            if ((n & 1) == 1) {
                count++;
            }
            n >>>= 1;
        }
        return count;
    }
}
```

### Why it works
- `n & 1` checks the least significant bit.
- `n >>>= 1` shifts the number right logically, bringing the next bit into position.

### Brute Force Complexity
- Time Complexity: O(32) = O(1), since the integer has a fixed number of bits.
- Space Complexity: O(1)

---

## 2. Best Approach: Bit Trick

### Why we use `n & (n - 1)`
This technique removes the rightmost set bit in `n`.

If `n` is nonzero:
- `n - 1` flips all bits after the lowest `1` bit, including that `1`.
- `n & (n - 1)` clears the lowest set bit.

This is very efficient when we only need to count set bits.

### Optimized Java Solution

```java
class Solution {
    public int hammingWeight(int n) {
        int count = 0;
        while (n != 0) {
            n &= (n - 1);
            count++;
        }
        return count;
    }
}
```

### Why this is better
- Each loop iteration removes one set bit.
- The number of iterations equals the number of `1` bits, not the total number of bits.

### Optimized Complexity
- Time Complexity: O(k), where `k` is the number of set bits.
- Space Complexity: O(1)

## 3. Follow-up: Many Calls Optimization

If this function is called many times, there are two good optimizations:

1. **Use a lookup table** for 8-bit chunks. Split the integer into bytes and sum precomputed results.
2. **Use built-in intrinsics** if the language provides them (`Integer.bitCount(n)` in Java).

Both methods are useful when you need repeated calls with high throughput.

---

## 4. Constraints

- `1 <= n <= 2^31 - 1`

Because `n` is a 32-bit integer, the basic bit trick is already fast and constant time.

## 5. Example

### Input
`n = 2147483645`

### Output
`30`

### Explanation
The binary form is `1111111111111111111111111111101`, which has thirty `1` bits.

## 6. Important Notes for Revision

- This is a classic bit manipulation problem.
- Use `n & (n - 1)` to remove the lowest set bit.
- The problem is best solved in constant space and with a small number of operations.
- For repeated calls, a lookup table or built-in bit count method is ideal.
