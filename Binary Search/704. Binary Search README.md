# 704. Binary Search

## Problem Summary

Given a sorted array of integers `nums` and a target value `target`, return the index of `target` if it exists in the array. Otherwise, return `-1`.

You must solve the problem using binary search.

Example:
- Input: `nums = [-1,0,3,5,9,12]`, `target = 9`
- Output: `4`

---

## Your Implementation
The local solution file is:
- `704. Binary Search.java`

Your Java solution uses the standard binary search approach:
- keep left and right pointers
- compute the middle index
- move the left or right pointer based on whether the middle value is smaller or larger than the target
- return the middle index when found

### Your Java code
```java
class Solution {
    public int search(int[] nums, int target) {
        int l = 0, r = nums.length-1, m = 0;
        while(l <= r){
            m = l + (r-l) / 2;
            if(nums[m] < target){
                l = m + 1;
            }else if(nums[m] > target){
                r = m - 1;
            }else{
                return m;
            }
        }
        return -1;
    }
}
```

### Approach, time, and space
- Approach: binary search on a sorted array using left and right pointers.
- Time Complexity: O(log n)
- Space Complexity: O(1)

## 1. Brute Force Approach

### Idea
Check every element in the array one by one until the target is found.

### Brute Force Code (Java)
```java
class Solution {
    public int search(int[] nums, int target) {
        for (int i = 0; i < nums.length; i++) {
            if (nums[i] == target) return i;
        }
        return -1;
    }
}
```

### Brute Force Complexity
- Time Complexity: O(n)
- Space Complexity: O(1)

---

## 2. Best Approach: Binary Search

### Why we use binary search
The array is sorted, so we can eliminate half of the search space at each step.

That means:
- if the middle element is smaller than the target, search the right half
- if the middle element is larger than the target, search the left half
- if it matches, return the index

### Why this is better
- It is much faster than checking every element.
- It works well for large sorted arrays.
- It is one of the most important interview patterns.

### Important note
Your implementation is the better approach for this problem because it uses binary search instead of a linear scan. It runs in O(log n) time and uses O(1) extra space.

- The array is sorted in ascending order.

These constraints make binary search a natural and efficient choice.

## 5. Important Notes for Revision

- Binary search only works on sorted arrays.
- Always update the left or right pointer based on the middle value.
- The search range is halved at every step.
- This is a classic interview problem and a core algorithm pattern.
